offline / 离线 Docker 构建说明
================================

conda create -n zs-rag python=3.12 -y
conda activate zs-rag
cd backend
./scripts/download_offline_wheels.sh

1. 在有网络的机器上准备依赖。Python 版本须与 backend/Dockerfile 一致（当前为 3.12），
   否则 pip 可能下载 grpcio 的源码包并尝试本地编译，依赖 g++（出现 'c++' 找不到或
   Getting requirements to build wheel 失败）。可在 backend 目录执行：

   ./scripts/download_offline_wheels.sh

   若系统默认是 python3.13，请显式使用 3.12，例如：

   PIP_DOWNLOAD_PYTHON=/path/to/python3.12 ./scripts/download_offline_wheels.sh

   或手动：

   python3.12 -m pip download -r requirements.txt -r requirements-docker-local-extras.txt -d offline_deps

2. 将生成的 offline_deps/*.whl（及少量 .tar.gz）随仓库或制品一并拷贝到构建环境。

3. 构建镜像：

   docker build -f Dockerfile.local --target development -t zs-rag-backend:dev .
   docker build -f Dockerfile.local --target production -t zs-rag-backend:prod .

   若使用 docker compose，且构建阶段不能访问 PyPI（与默认 Dockerfile 中
   pip install -r requirements.txt 失败时），请改用离线 Dockerfile：

   BACKEND_DOCKERFILE=Dockerfile.local docker compose -f docker-compose.yml up -d --build

   或在项目根目录 .env 中设置：BACKEND_DOCKERFILE=Dockerfile.local

4. 在 macOS/Windows 上为 Linux 容器准备依赖时，建议在 Linux 容器或 CI 中执行
   pip download，或使用与目标 manylinux 匹配的 --platform/--python-version，
   避免出现无法安装的二进制包。
