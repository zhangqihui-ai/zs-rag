offline / 离线 Docker 构建说明
================================

conda create -n zs-rag python=3.12 -y
conda activate zs-rag
cd backend
./scripts/download_offline_wheels.sh

1. 在有网络的机器上准备依赖。Python 版本须与 backend/Dockerfile 一致（当前为 3.12），
   否则 pip 可能下载 grpcio 的源码包并尝试本地编译，依赖 g++（出现 'c++' 找不到或
   Getting requirements to build wheel 失败）。可在 backend 目录执行：

   ./scripts/download_offline_wheels.sh

   若系统默认是 python3.13，请显式使用 3.12，例如：

   PIP_DOWNLOAD_PYTHON=/path/to/python3.12 ./scripts/download_offline_wheels.sh

   或手动：

   python3.12 -m pip download -r requirements.txt -r requirements-docker-local-extras.txt -d offline_deps

   升级依赖后（例如 pymilvus 2.4.x → 2.5.6），建议先清理旧 wheel 再下载，避免离线
   构建仍安装旧版本：

   rm -f offline_deps/pymilvus-*.whl offline_deps/grpcio-*.whl
   ./scripts/download_offline_wheels.sh

2. 关键版本（与 requirements.txt 一致，供离线包核对）：

   - pymilvus==2.5.6   （对齐 docker-compose milvus v2.5.6；LightRAG 需 list_databases）
   - lightrag-hku>=1.4.0,<1.5
   - neo4j==5.27.0
   - opendataloader-pdf>=1.0.0   （PDF 默认解析；需 JRE 11+，Docker 镜像已装 default-jre-headless）

3. 将生成的 offline_deps/*.whl（及少量 .tar.gz）随仓库或制品一并拷贝到构建环境。
   同目录下需包含 offline_deps/jre/temurin-21-jre.tar.gz（离线 JRE，避免构建时访问 deb.debian.org）：
   ./scripts/download_offline_jre.sh
   或随 download_offline_wheels.sh 一并下载（默认会自动拉 JRE）。

4. 构建镜像（离线，无需 apt 外网；**Docker build context 必须是 backend 目录**）：

   # 在仓库根目录：
   docker build -f backend/Dockerfile.local --target development -t zs-rag-backend:dev backend

   # 在 backend 目录内：
   docker build -f Dockerfile.local --target development -t zs-rag-backend:dev .

   docker compose（context 已是 ./backend）：
   BACKEND_DOCKERFILE=Dockerfile.local docker compose up -d --build backend

   若构建机可访问国内 Debian 镜像但无法访问 deb.debian.org，在线 Dockerfile 可设：
   APT_MIRROR=http://mirrors.aliyun.com/debian docker compose build backend

5. 在 macOS/Windows 上为 Linux 容器准备依赖时，建议在 Linux 容器或 CI 中执行
   pip download，或使用与目标 manylinux 匹配的 --platform/--python-version，
   避免出现无法安装的二进制包。
