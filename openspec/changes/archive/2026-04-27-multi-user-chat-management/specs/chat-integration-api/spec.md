## ADDED Requirements

### Requirement: RESTful Chat API
The system SHALL expose RESTful API endpoints for creating, retrieving, updating, and deleting chat sessions and their configurations.

#### Scenario: Creating a chat via REST
- **WHEN** a client sends a POST request to `/api/v1/chats` with valid authentication
- **THEN** the system returns a new chat session ID for the authenticated user and `enterprise_space`.

### Requirement: Real-time WebSocket API
The system SHALL support WebSocket connections to facilitate real-time streaming of LLM responses.

#### Scenario: Streaming a response via WebSocket
- **WHEN** a user sends a prompt over an established WebSocket connection for an active session
- **THEN** the server MUST stream the token chunks back to the client as they are generated by the LLM.

### Requirement: Embeddable Chat UI
The system SHALL provide an embeddable script or iframe endpoint allowing third-party systems to integrate the chat interface.

#### Scenario: Rendering embedded chat
- **WHEN** a third-party application includes the embed code snippet
- **THEN** a simplified, un-branded version of the chat UI is rendered, communicating via the provided APIs.
