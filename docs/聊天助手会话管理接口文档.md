# 聊天对话 & 会话管理接口文档

> **对外接入速览**（认证、15 个接口一览、curl）：请参阅 [`聊天对话对外API接入文档.md`](./聊天对话对外API接入文档.md)。  
> **多轮历史如何实现**（`session_id`、`messages` / `question`、DB 20 条窗口）：请参阅 [`会话多轮历史实现说明.md`](./会话多轮历史实现说明.md)。

本文档描述 **zs-rag** 后端中与 **对话（Chat）**、**会话（Session）**、**消息（Message）**、**OpenAI 形态补全**、**流式聊天（SSE）** 相关的 HTTP 接口，便于第三方系统通过 REST + SSE 嵌入能力。**实现源码**：`backend/app/api/router.py`（挂载前缀）、`backend/app/api/routes/chats.py`、`backend/app/api/routes/chat_completions.py`；数据模型见 `backend/app/schemas/chat.py`。

## 公共约定

| 项目 | 说明 |
|------|------|
| **Base URL** | 例：`https://<你的域名>` 或 `http://localhost:8000` |
| **完整前缀** | 对话与会话：`{BASE_URL}/api/v1/chats`；补全：`{BASE_URL}/api/v1/chat` |
| **认证** | `Authorization: Bearer <access_token>` 或嵌入 Key `zs_rag_embed_...` |
| **企业空间** | `X-Enterprise-Space: <slug>`；**嵌入 Key 可不传**（按 Key 自动识别）；JWT 省略时为 `default` |
| **Content-Type** | 除流式接口外，请求体使用 `application/json` |
| **成功响应** | 由 FastAPI **直接序列化 Pydantic 模型**为 JSON（一般为 snake_case），**不是**全局 `{code,data,message}` 包裹 |
| **业务错误** | HTTP 4xx/5xx，响应体为 `{ "code", "message", "request_id", "details?" }`（见 `app.core.errors`） |
| **参数校验** | 请求体不符合 schema 时多为 **422**，细节以 FastAPI 校验输出为准 |
| **主键形态** | `chat_id`、`session_id`、消息 `id` 均为 **字符串**（服务端 UUID，典型长度 36） |
| **术语** | **Chat（对话）**：顶层容器，模型与知识库等配置挂在该资源上；其下多个 **Session（会话）** 共享同一份配置 |
| **数据库迁移** | 若执行 `016_chat_uuid` 等破坏性迁移，会 **清空** 既有聊天表数据，部署前请评估 |

环境变量示例（下文 `{CHAT}`、`{SESS}` 请替换为接口返回的真实 id）：

```bash
BASE_URL="http://localhost:8000"
TOKEN="<登录后获得的 access_token>"
SPACE="default"
```

## 接口一览

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/v1/chats` | 创建对话（响应 `201`）；服务端**同时**创建首个默认会话，标题为 **「会话 1」** |
| GET | `/api/v1/chats` | 对话列表，`skip`、`limit` 分页 |
| GET | `/api/v1/chats/{chat_id}` | 获取单个对话 |
| PUT | `/api/v1/chats/{chat_id}` | 修改标题，请求体 `ChatConversationUpdate` |
| DELETE | `/api/v1/chats/{chat_id}` | 删除对话，**204** |
| GET | `/api/v1/chats/{chat_id}/sessions` | 某对话下会话列表 |
| POST | `/api/v1/chats/{chat_id}/sessions` | 在该对话下创建会话，**201**；请求体 `ChatSessionCreate`，`title` 默认 **「新会话」** |
| GET | `/api/v1/chats/sessions/{session_id}` | 会话详情 |
| PUT | `/api/v1/chats/sessions/{session_id}` | 修改会话标题；请求体 `ChatSessionUpdate`（`title` 必填） |
| DELETE | `/api/v1/chats/sessions/{session_id}` | 删除会话，**204**；若为该对话**最后一个**会话，可能一并删除对话 |
| GET | `/api/v1/chats/sessions/{session_id}/config` | 读模型配置（对话级） |
| PUT | `/api/v1/chats/sessions/{session_id}/config` | 写模型配置（对话级），`ChatConfigurationUpdate` |
| GET | `/api/v1/chats/sessions/{session_id}/messages` | 消息历史，`skip`、`limit` |
| POST | `/api/v1/chats/sessions/{session_id}/stream` | 内置 SSE，`ChatStreamRequest`：`{ "content": "..." }` |
| POST | `/api/v1/chat/completions` | OpenAI 形态补全，见第三节 |

**路径注意**：`chat_id` **不能**为字面量 `sessions`（与 `/chats/sessions/...` 区分）。路由将 **`/sessions/{session_id}`** 注册在 **`/{chat_id}`** 之前，避免误匹配。

## 一、对话（Chat）— `/api/v1/chats`

### 创建对话 — `POST /api/v1/chats`

请求体 `ChatConversationCreate`：

| 字段 | 类型 | 说明 |
|------|------|------|
| `title` | string | 默认 `新建对话` |
| `configuration` | object? | 可选；字段同配置：`model_provider`、`model_name`、`knowledge_base_ids`、`temperature`、`max_tokens`、`top_p` |

```json
{
  "title": "新建对话",
  "configuration": {
    "model_name": "deepseek-chat",
    "model_provider": "deepseek",
    "knowledge_base_ids": [1, 2],
    "temperature": 0.7,
    "max_tokens": 2000,
    "top_p": 1.0
  }
}
```

响应 **201**，体为 `ChatConversationResponse`：`id`（即 `chat_id`）、`user_id`、`enterprise_space_id`、`title`、`created_at`、`updated_at`、`model_provider`、`model_name`、`knowledge_base_ids`、`temperature`、`max_tokens`、`top_p`。

```bash
curl -sS -X POST "${BASE_URL}/api/v1/chats" \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "X-Enterprise-Space: ${SPACE}" \
  -H "Content-Type: application/json" \
  -d '{"title":"我的项目讨论"}'
```

### 列表 — `GET /api/v1/chats?skip=0&limit=100`

### 获取 / 修改 / 删除

- **GET** `/api/v1/chats/{chat_id}` → `ChatConversationResponse`
- **PUT** `/api/v1/chats/{chat_id}` → 请求体 `{"title":"新标题"}` → `ChatConversationResponse`
- **DELETE** `/api/v1/chats/{chat_id}` → **204**

### 对话下的会话

- **GET** `/api/v1/chats/{chat_id}/sessions` → `ChatSessionResponse[]`
- **POST** `/api/v1/chats/{chat_id}/sessions` → 请求体 `{"title":"会话 2"}`（可省略，默认「新会话」）→ **201** `ChatSessionResponse`

```bash
curl -sS "${BASE_URL}/api/v1/chats/${CHAT}/sessions" \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "X-Enterprise-Space: ${SPACE}"
```

**`ChatSessionResponse`** 字段：`id`、`chat_id`、`user_id`、`enterprise_space_id`、`title`、`created_at`、`updated_at`（JSON 中为 `chat_id`，对应 ORM 的 `conversation_id`）。

## 二、会话子资源 — `/api/v1/chats/sessions/{session_id}`

配置存于**对话（chat）**行；通过任意隶属该对话的 `session_id` 均可读写同一份配置。

### 会话 CRUD

见上一节「接口一览」。

### 配置 — `GET|PUT .../config`

- **GET** → `ChatConfigurationResponse`：`id`、`chat_id`（均为对话 UUID，值相同）、`model_provider`、`model_name`、`knowledge_base_ids`、`temperature`、`max_tokens`、`top_p`
- **PUT** → 请求体与上同，字段均可按需部分提交（未传字段由服务端按更新语义处理）

### 消息 — `GET .../messages`

查询参数：`skip`、`limit`（默认 0、100）。响应：`ChatMessageResponse[]`：`id`、`session_id`、`role`、`content`、`created_at`。

### 内置 SSE — `POST .../stream`

请求体 `ChatStreamRequest`：`{ "content": "用户本轮纯文本" }`（`content` 至少 1 字符）。

响应 **`Content-Type: text/event-stream`**；每行 `data: ` + 单行 JSON + `\n\n`。本接口仍使用**历史事件名**（非 OpenAI chunk）：

| `type` / 形态 | 含义 |
|---------------|------|
| `assistant_delta` | 字段 `content` 为增量文本 |
| `assistant_done` | 含落库后的 `id`、`session_id`、`role`、`content`、`created_at` |
| `type: "error"` | 异常时，`message` 为说明 |

```bash
curl -N -sS -X POST "${BASE_URL}/api/v1/chats/sessions/${SESS}/stream" \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "X-Enterprise-Space: ${SPACE}" \
  -H "Content-Type: application/json" \
  -d '{"content":"请用三句话介绍 RAG。"}'
```

## 三、OpenAI 形态补全 — `POST /api/v1/chat/completions`

单数路径 **`/api/v1/chat`**，子路径 **`/completions`**。

### 行为

| 条件 | 行为 |
|------|------|
| 有 `chat_id`，无 `session_id` | 使用对话配置，**新建会话**（标题形如「会话 *n*」），在本会话中落库并生成回复 |
| 有 `chat_id` 与 `session_id` | 校验会话属于该对话后，在已有会话中续聊 |
| `stream: true` | SSE：帧为 **`chat.completion.chunk`** 结构，并附加 **`chat_id`、`session_id`**；流结束后一行 **`data: [DONE]`** |
| `stream: false` | JSON：**`chat.completion`** 结构，并附加 **`chat_id`、`session_id`** |

### 请求体 `ChatCompletionsRequest`

| 字段 | 必填 | 说明 |
|------|------|------|
| `chat_id` | 是 | 对话 id，非空字符串 |
| `session_id` | 否 | 省略则自动建会话 |
| `stream` | 否 | 默认 `false` |
| `messages` | * | OpenAI 风格 `{ "role", "content" }`；**仅** `system` / `user` / `assistant` 且 `content` 非空（空白会跳过）会进入模型上下文 |
| `question` | * | 当**没有**可用规范化后的 `messages` 时：取会话内最近 **20** 条 DB 消息 + 本条 user |

\* **校验**：须至少满足——`messages` 为非空**数组**（允许元素全无效时再依赖 `question`），或 `question` 去掉空白后非空。若传了 `messages` 但规范化后为空且 `question` 也无有效内容 → **400** `MESSAGES_NORMALIZATION_EMPTY`。若规范化成功但没有任何 `user` 有效内容 → **400** `INVALID_MESSAGES`。

其它 JSON 字段（如 OpenAI 的 `model`）会被 **`extra="ignore"`** 忽略；实际模型以对话配置为准。

### 流式示例

```bash
curl -N -sS -X POST "${BASE_URL}/api/v1/chat/completions" \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "X-Enterprise-Space: ${SPACE}" \
  -H "Content-Type: application/json" \
  -d '{
    "chat_id": "'"${CHAT}"'",
    "stream": true,
    "messages": [
      {"role": "user", "content": "请用一句话介绍 RAG。"}
    ]
  }'
```

流内帧为 JSON：`id`（`chatcmpl-*`）、`object: chat.completion.chunk`、`created`、`model`、`choices[0].delta` / `finish_reason`，以及 **`chat_id`、`session_id`**。末帧常为 `finish_reason: "stop"` 且 `delta` 为空；随后：

```text
data: [DONE]
```

流式过程中若生成器异常，可能下发：`data: {"error":{"message":"...","type":"internal_error"}}`（非统一业务错误包装）。

### 非流式响应示例

HTTP 200，体与代码中 `JSONResponse` 一致，例如：

```json
{
  "id": "chatcmpl-0123456789abcdef01234567",
  "object": "chat.completion",
  "created": 1714032000,
  "model": "deepseek-chat",
  "choices": [
    {
      "index": 0,
      "message": { "role": "assistant", "content": "完整回复文本" },
      "finish_reason": "stop"
    }
  ],
  "chat_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "session_id": "b2c3d4e5-f6a7-8901-bcde-f12345678901"
}
```

### 非流式 + `question` + 已有 `session_id`

```bash
curl -sS -X POST "${BASE_URL}/api/v1/chat/completions" \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "X-Enterprise-Space: ${SPACE}" \
  -H "Content-Type: application/json" \
  -d '{
    "chat_id": "'"${CHAT}"'",
    "session_id": "'"${SESS}"'",
    "stream": false,
    "question": "继续上文，给出三点建议。"
  }'
```

## 四、业务错误码（节选）

与聊天相关的 `AppError.code`（响应 HTTP 状态与统一 `code`、`message` 字段）：

| HTTP | `code` | 说明 |
|------|--------|------|
| 404 | `CONVERSATION_NOT_FOUND` | 对话不存在或无权访问 |
| 404 | `SESSION_NOT_FOUND` | 会话不存在或无权访问 |
| 400 | `SESSION_CHAT_MISMATCH` | `session_id` 不属于所传 `chat_id` |
| 400 | `INVALID_MESSAGES` | `messages` 规范化后无有效 **user** 内容 |
| 400 | `MESSAGES_NORMALIZATION_EMPTY` | `messages` 无效且未提供有效 `question` |
| 500 | `COMPLETION_FAILED` | 非流式补全未捕获异常（带错误信息） |

## 五、未单独提供的路由

- 批量删除会话：需多次 `DELETE /api/v1/chats/sessions/{session_id}` 或由网关聚合。
- 按 id 删除单条消息：当前无独立 REST（以前端本地删除或后续迭代为准）。

## 六、排错与其它

| 现象 | 建议 |
|------|------|
| `401` | Token 无效或过期 |
| `422` | 请求体不符合 schema（如补全未传 `messages` 数组也未传 `question`） |
| SSE 缓冲 | 网关需支持流式；响应已设 `Cache-Control: no-cache`、`X-Accel-Buffering: no` |
| 无模型输出 | 检查对话/配置中 `model_name`、`model_provider` 及模型管理是否启用 |
